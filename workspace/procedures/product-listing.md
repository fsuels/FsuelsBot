## Phase 1: Product Discovery on 1688

### 1.1 Vendor Quality Checks (CRITICAL)

**Before selecting ANY product, verify the vendor:**

- [ ] **Store rating** ‚Äî 4.5+ stars preferred, never below 4.0
- [ ] **Transaction volume** ‚Äî Active sales (shows recent orders)
- [ ] **Store age** ‚Äî Established sellers (1+ years preferred)
- [ ] **Response rate** ‚Äî High response = reliable communication
- [ ] **Product availability** ‚Äî Check stock indicators, avoid "last item" or low stock
- [ ] **1-piece dropshipping** ‚Äî Must allow single unit orders (no MOQ)
- [ ] **Fast warehouse delivery** ‚Äî 24-48hrs to BuckyDrop China warehouse

**Red flags (AVOID these vendors):**
- ‚ùå No recent sales/reviews
- ‚ùå Store opened recently with no history
- ‚ùå Poor ratings or many complaints
- ‚ùå Listings that look too good to be true
- ‚ùå Products showing "sold out" or "discontinued"
- ‚ùå **Old listings** ‚Äî factories keep discontinued products as "hooks"

### 1.2 Product Selection Criteria

**We sell: Mommy and me / family matching outfits**

Good products have:
- ‚úÖ **Multiple sizes** ‚Äî Adult AND child sizes in same listing
- ‚úÖ **Matching designs** ‚Äî Same fabric/pattern for both
- ‚úÖ **Good photos** ‚Äî Clear, professional images
- ‚úÖ **Hot market appeal** ‚Äî Seasonal (Valentine, Easter, etc.) or evergreen (solid patterns)
- ‚úÖ **Reasonable base cost** ‚Äî Allows 50%+ margin at competitive price

**Avoid:**
- ‚ùå Adult-only products (not "matching")
- ‚ùå Poor quality photos
- ‚ùå Products from unreliable vendors
- ‚ùå Items already in our catalog (check Phase 0!)

### 1.3 Freshness Check (CRITICAL)

- Sort by "newest" or "recent sales" when searching
- **Check listing creation date** ‚Äî recently created listings only
- Verify product photos look current (not dated styles)
- **Avoid old listings** ‚Äî factories don't update discontinued products, keep old listings as "hooks" to attract buyers

**State:** "Product discovery: Vendor [name] has [rating], [transaction count] sales. Listing created [date]. 1-piece OK. 24-48hr delivery. Product fits matching criteria."

```
MINIMUM PRICE = TOTAL COST √ó 2
```

## Phase 4: Shopify Draft Creation

### 4.1 Basic Info
- **Title:** Clear, SEO-friendly (e.g., "Valentine Heart Mommy and Me Matching Pajamas")
- **Description:** Benefits, materials, sizing info
- **Product type:** Set appropriately (T-Shirt, Sweatshirt, Dress, Pajamas, etc.)

### 4.2 Category & Metafields (CRITICAL FOR DISCOVERABILITY)

**Study existing similar products** to understand the tagging patterns!

**Category Selection:**
- Choose correct category (e.g., Sweatshirts in Clothing > Tops)
- This auto-populates some category metafields

**Category Metafields (for site filters):**
- **Color** ‚Äî Use exact values from existing filter (Red, Pink, White, Black, Blue, Green, etc.) ‚Äî capitalize consistently!
- **Size** ‚Äî All sizes in the listing
- **Fabric** ‚Äî Material type
- **Age group** ‚Äî Kids, Adults, All Ages
- **Target gender** ‚Äî Female, Male, Unisex

**Product Metafields (for breadcrumbs/SEO):**
- **Pattern** ‚Äî Hearts, Floral, Solid, Stripes, etc.
- **Style** ‚Äî Casual, Formal, etc.
- **Type** ‚Äî Product type
- **SubCategory / SubCategory2** ‚Äî Navigation hierarchy
- **Category1** ‚Äî Top-level category

### 4.3 Tags (MUST INCLUDE)

Tags enable filtering and collection organization. **Study existing products for patterns!**

**Required tags:**
- **Color tags** ‚Äî `red heart`, `pink`, `white`, etc.
- **Size tags** ‚Äî If needed for filtering
- **Collection tags** ‚Äî `valentines day`, `christmas`, `summer`, etc.
- **Relationship tags** ‚Äî `mother daughter`, `daddy and me`, `matching family`
- **Brand tags** ‚Äî `Mommy and Me`
- **Product type tags** ‚Äî `sweatshirt`, `pajamas`, `t-shirt`, etc.

**Example for Valentine product:**
```
valentines day, red heart, mother daughter, Mommy and Me, matching family, heart sweatshirt, family outfit
```

### 4.4 Collections

Add product to appropriate collections:
- **Mommy and Me** ‚Äî always
- **New Mommy & Me** ‚Äî if new product
- **Seasonal collection** ‚Äî Valentine's Day, Christmas, Summer, etc.
- **Category collections** ‚Äî Swimwear, Pajamas, T-Shirts, etc.

### 4.5 Images
- Upload all product images
- **FACE CHECK:** If Chinese faces visible ‚Üí need face swap
- Order: Main image first, then angles, then size chart

### 4.6 Face Swapping (if needed)

**When:** Product photos show Chinese models

**Process:**
1. Download product images
2. Go to: https://aifaceswap.io/#face-swap-playground
3. Use faces from: `C:\Users\Fsuels\Downloads\faces`
4. Swap to American-looking faces
5. Save new images
6. Upload to Shopify

**State:** "Face swap: [X] images processed with American faces."

### 4.7 Size Chart (CRITICAL)

**Get measurements FROM the 1688 listing:**

1. **Extract size table from 1688** ‚Äî find the measurements (chest, length, sleeve, etc.)
2. **Create table with actual numbers** ‚Äî use the exact measurements provided
3. **Convert size names to our standard:**
   - Kids: 80cm, 90cm, 100cm, 110cm, 120cm, 130cm, 140cm, 150cm
   - Adults: Adult S, Adult M, Adult L, Adult XL, Adult XXL, Adult 3XL
4. **Apply size conversion script** ‚Äî format for Shopify
5. **Verify dynamic selector works** ‚Äî customer picks size, sees correct measurements

**Never guess sizes** ‚Äî always use the actual numbers from the 1688 listing.

**State:** "Size chart: Extracted from 1688, [X] sizes, converted to our naming, script applied."

### 4.8 Variants & Pricing

- Set up all size variants (Adult S/M/L/XL, Child sizes)
- Apply calculated price to all variants
- Verify no variant is priced below minimum

### 4.9 Save as DRAFT

- **NEVER publish active** ‚Äî Francisco reviews all drafts first
- Save as DRAFT
- Note draft number for tracking

**State:** "Draft #[N] created: [Product name]. Price: $[X]. Tags: [list]. Collections: [list]. Awaiting Francisco's review."
